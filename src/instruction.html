Manual

KROHA.JS is an emulator of a first-generation von Neumann machine (computer) with a CISC architecture.

The computer has 5 panels:

The main window with memory, where you move the cursor and enter data.

An information window that shows the current memory cell and the state of the computer, with the following fields:

  • Mode - the current mode of the computer
  • Cell - the memory cell the cursor is in
  • Decimal - the decimal value of the memory cell the cursor is in
  • Instruction - the instruction written in the cell the cursor is in
  • Operand Value - the address fields A1, A2, A3 of the cell (Operand) and the decimal values stored at those addresses (Value)

A reference panel with a short description of each instruction and its code, the hotkeys, and the Program toolbar.

The arithmetic logic unit (ALU), which shows the state of the instruction register (IR), the accumulator (AC) and the program counter (PC).

The Screen, where the HALT instruction outputs the selected cells.

The computer does not distinguish between data and instructions — the user (programmer) decides what each cell means

Use the arrow keys to move the cursor through the memory cells. The 0 and 1 keys write the corresponding bit; Backspace writes a 0.

The 2 (up), 4 (left), 6 (right) and 8 (down) keys also move the cursor — convenient on a numeric keypad or a mobile keyboard.

On large screens the toolbar is in the Commands panel; on small screens it sits under the memory table.

  • Examples... - loads a ready-made demo program into memory
  • Speed - how fast Auto mode executes; the rightmost position runs instantly
  • Share - copies a link with the current program encoded in the URL, so you can send it to someone else
  • Clear - resets all memory cells to zero

The program in memory is saved in your browser on every edit and restored on your next visit

The computer works in 3 modes: automatic (Auto), step-by-step (Step) and clock-by-clock (Tact). In Auto mode the computer executes instructions one after another until it reaches the HALT instruction; the Speed slider sets the pace, and pressing Auto again pauses and resumes execution. In Step mode the computer executes one instruction at a time. In Tact mode each instruction is executed clock by clock, and the information window explains what is happening

Keys responsible for starting the execution mode:

  • A - Auto
  • S - Step
  • T - Tact

Editing memory (with the 0, 1 or Backspace keys) or pressing E switches the computer back to editor mode: the state is reset and execution stops

On mobile, use the three buttons on the page to start the corresponding mode

After a HALT message (or an error such as "ERROR Overflow", "ERROR Division by zero" or "ERROR Infinite loop"), press any run key (A, S, T) to reset the machine (ALU and screen). Execution will not start — only the values are reset

Hotkeys work in any keyboard layout — on non-Latin layouts the physical position of the A, S, T and E keys is used. Keys are not case sensitive

KROHA.JS supports 8 instructions:

Copies the value from the cell at address A1 to the cell at address A3. A2 is unused

Instruction step by step

  • Step 1: Read the instruction at the PC address into the IR
  • Step 2: Increase the PC by 1 (prepare for the next instruction)
  • Step 3: Load the value at address A1 into the AC
  • Step 4: Write the AC to the cell at address A3

Adds the value in the cell at address A1 to the value in the cell at address A2 and writes the result to the cell at address A3

If the result is greater than 32767 (15 bits), the machine stops with an "ERROR Overflow" message

  • Step 5: Output the third number to the screen

The easiest way to see the machine in action: pick a program from the Examples... dropdown and press A (Auto).

For instance, "Add: 5 + 7" consists of two instructions and two numbers. Cell 0000 holds 001 1000 1001 1010 — ADD: take the value at address 1000 (which is 5), add the value at address 1001 (which is 7), and store the result at address 1010. Cell 0001 holds 111 1000 1001 1010 — HALT: stop and output the cells at those three addresses. Run it and the Screen shows 5, 7 and 12.

Try running it in Tact mode (T) to watch every clock cycle of each instruction, or load the other examples to see jumps and loops.
